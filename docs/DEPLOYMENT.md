# Deployment Guide

Deploy Aria as Jeff's daily AI workstation on a single Linux machine.

## Layout

```
AI/                          # AI_ROOT (platform data)
  Aria/                      # assistant + GUI + workstation CLI
  AI-Platform/               # platform services, compose, applications
  Data/                      # platform runtime data (created by init)
  compose/                   # docker compose (generated by aiplatform init)
```

## One-command lifecycle

| Action | Command |
|--------|---------|
| Install | `./workstation install` |
| Configure | `./workstation configure` |
| Start | `./workstation start` |
| Stop | `./workstation stop` |
| Restart API | `./scripts/jarvis-ctl.sh restart` |
| Update | `./workstation update` |
| Backup | `./workstation backup` |
| Restore | `./workstation restore backups/workstation_*.tar.gz` |
| Verify | `./workstation verify` |
| Doctor | `./workstation doctor` |
| Full status | `./scripts/jarvis-ctl.sh status --full` |

## Services

Aria manages components via the workstation registry:

```bash
./workstation status          # JSON health snapshot
./workstation up              # bootstrap autostart services
./workstation up --profile data   # start platform docker profile
./workstation diagnose
./workstation recover
./workstation list
```

Core components: Ollama, Aria HTTP server, proactive scheduler, optional docker services (postgres, redis, litellm, qdrant, …).

## Desktop autostart

```bash
./scripts/install-systemd-user.sh   # start on login
```

## LAN access

```bash
./scripts/enable-lan.sh             # bind 0.0.0.0 + API key
./scripts/harden-security.sh          # audit exposure
```

## Hardware profiles

Machine-specific tuning scripts live under `scripts/`:

- `optimize-rx7600-8gb.sh` — AMD RX 7600 8GB
- `install-cuda-pytorch.sh` / `install-rocm-pytorch.sh` — GPU PyTorch
- `enable-nvidia-gpu.sh` — dual GPU (AMD display + NVIDIA compute)

Run `./scripts/workstation-hardware.sh` for a detected-hardware summary (when available).
